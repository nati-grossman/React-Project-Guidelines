למה זה חשוב?

קונבנציה אחידה מונעת בלגן ומקצרת זמן חיפוש.

  • קריאות: שמות משקפים במדויק את התוכן.
  • אחידות: כולם עובדים לפי אותו סגנון.
  • ייבוא נקי: משתלב מצוין עם Barrel Exports.

קונבנציות מקובלות ב־React

  • קומפוננטות: PascalCase — למשל UserCard.tsx, UserList.tsx.
  • Hooks: תמיד prefix של use וב־camelCase — למשל useUsers.ts, useAuth.ts.
  • Services: camelCase או PascalCase — למשל userService.ts, AuthService.ts.
  • CSS / Styles: kebab-case או בהתאמה לשם הקומפוננטה — למשל user-card.css, dashboard.css.
  • Constants: UPPER_SNAKE_CASE — למשל API_URL.ts, MAX_RETRIES.ts.
  • Test files: שם הקובץ + סיומת .test — למשל UserCard.test.tsx.

טיפים פרקטיים

  • קומפוננטות: תמיד PascalCase.
  • Hooks: תמיד להתחיל ב־use.
  • שם תואם לתוכן: קבצים/תיקיות משקפים את מה שבפנים.
  • אחידות מעל הכל: בחרו סגנון אחד והיצמדו אליו.

סיכום

שמירה על Naming Conventions ברורים ועקביים משפרת קריאות, תחזוקה ושיתוף קוד. בשילוב עם Feature-Based Structure ו־Barrel Exports, הקוד נשאר נקי, מודולרי וקל להבנה.
