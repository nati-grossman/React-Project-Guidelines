Feature-Based Folder Structure – מבנה לפי פיצ׳רים

← חזרה לדף הבית

למה חשוב סדר ומבנה תקין?

בפרויקטים גדולים, סדר בתיקיות הוא ההבדל בין פרויקט שזורם לפרויקט תקוע.

  • קל יותר לאתר קבצים ולהבין את הזרימה.
  • תחזוקה ושינויים מהירים בלי לשבור חלקים אחרים.
  • צמיחה בריאה של הקוד מבלי להעמיס על מבנה הפרויקט.

המספרים של הסדר

מבנה לפי סוג קובץ (Components/Services/Hooks/Utils) נוח בהתחלה, אבל ברגע שמתרחבים הוא יוצר עומס:

  • תיקיות מתנפחות עם עשרות קבצים.
  • הקשר בין קבצים שייך לפיצ׳ר הולך לאיבוד.
  • ניווט ארוך וייבוא מסורבל בין אזורים שונים.

הפתרון: מבנה לפי פיצ׳רים

Feature-Based Folder Structure (מבנה לפי פיצ׳רים) אומר שכל פיצ׳ר של המערכת מקבל תיקייה משלו, שבתוכה נמצאים כל הקבצים הקשורים אליו: קומפוננטות, Services, Hooks, Styles, Tests – הכל במקום אחד.

כך נשמר ההקשר העסקי והטכני של כל פיצ׳ר במקום ממוקד וקל לניהול.

דוגמה:

src/
├─ features/
│  ├─ users/
│  │  ├─ components/
│  │  │  ├─ UserCard.tsx
│  │  │  └─ UserList.tsx
│  │  ├─ hooks/
│  │  │  └─ useUsers.ts
│  │  ├─ services/
│  │  │  └─ userService.ts
│  │  ├─ styles/
│  │  │  └─ users.css
│  │  └─ index.ts
│  ├─ dashboard/
│  │  └─ ...
│  └─ settings/
│     └─ ...
├─ shared/
│  ├─ components/
│  └─ utils/
└─ App.tsx

יתרונות של מבנה לפי פיצ׳רים

  • קל למצוא קוד: כל מה שקשור ל־Users נמצא בתיקייה אחת, אין צורך לקפוץ בין כמה תיקיות.
  • תחזוקה פשוטה: כשיש באגים או שינויים בפיצ׳ר מסוים, עובדים בתוך תיקייה אחת בלבד.
  • Scalability: כשמוסיפים פיצ׳ר חדש – פשוט יוצרים תיקייה חדשה, בלי לבלגן תיקיות מרכזיות כמו Components או Services.
  • קוד מודולרי: מאפשר לייצא פיצ׳רים בצורה מודולרית, ולשתף אותם בין פרויקטים.
  • קל ל־index.ts: אפשר ליצור index.ts בכל תיקייה שמייצא את כל הקבצים שלה, וככה לייבא בקוד הראשי בצורה נקייה.

טיפים חשובים

  • Shared vs Features: כל מה שמשותף בין פיצ׳רים (כמו קומפוננטות כלליות או פונקציות Utilities) צריך להיות בתיקיית shared.
  • Index.ts: שימוש ב־index.ts בכל תיקייה עוזר להקל על ייבוא קבצים ומונע נתיבים ארוכים.
  • Keep it flat: בתוך הפיצ׳ר, לא כדאי ליצור היררכיות עמוקות מדי – עד 2–3 רמות מספיק לרוב.
  • Tests בצמוד לפיצ׳ר: אם יש קבצי בדיקה (unit/integration), שמרו אותם ליד הקוד שהם בודקים.
