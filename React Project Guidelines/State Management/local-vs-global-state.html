Local State vs Global State – מתי כל אחד מתאים

← חזרה לדף הבית

הבדל בסיסי

לפני שבוחרים Context/Redux/כלי אחר, חשוב להבין מתי state צריך להיות מקומי ומתי גלובלי.

Local State

state פנימי לקומפוננטה, מנוהל עם useState/useReducer, ונגיש לה ולצאצאים דרך props. 

// Local state בדוגמת טופס
function LoginForm() {
  const [email, setEmail] = React.useState('');
  const [open, setOpen] = React.useState(false); // modal מקומי
  // ...
}

דוגמאות

  • טופס עם ערכים זמניים (input, checkbox).
  • מצב פתוח/סגור של Modal.
  • טיימר פנימי בקומפוננטה אחת.

יתרונות

  • פשוט ומהיר לשימוש.
  • קל לניהול ולבדיקות.
  • מבודד — לא משפיע על חלקים אחרים.

Global State

state הזמין ממספר אזורים באפליקציה (Context, Redux, Zustand, Recoil וכו'). 

// דוגמה קצרה עם Context ל־Theme גלובלי
const ThemeContext = React.createContext('light');
function App() {
  return <ThemeContext.Provider value="dark"><Layout /></ThemeContext.Provider>;
}
function Title() {
  const theme = React.useContext(ThemeContext);
  return <h1 className={theme}>Hello</h1>;
}

דוגמאות

  • פרטי משתמש מחובר (user, token).
  • Theme (מצב Dark/Light).
  • הגדרות מערכת המשפיעות על כמה קומפוננטות.

יתרונות

  • שיתוף נתונים רחב.
  • חוסך Props Drilling.
  • נוח כאשר חלקים נפרדים צריכים להגיב לאותו שינוי.

מתי לבחור מה?

בחר Local State כאשר…

  • הנתונים רלוונטיים רק לקומפוננטה (או לצאצאים הישירים שלה).
  • יש עדכונים תכופים מקומיים לאזור אחד.

בחר Global State כאשר…

  • הנתונים נדרשים ממספר אזורים שונים.
  • ברצונך להימנע מ־Props Drilling עמוק.
  • יש צורך בתגובה רוחבית לשינויים בנתונים.

כלל אצבע + טיפ חשוב

  • התחל ב־Local State — פשוט יותר.
  • העבר ל־Global רק כשבאמת צריך לשתף מידע בין אזורים.
  • שמור את ה־state קרוב ככל האפשר למקום שבו הוא נדרש.

סיכום

  • Local State: מצוין למידע פנימי וממוקד.
  • Global State: מתאים לשיתוף מידע רחב.
