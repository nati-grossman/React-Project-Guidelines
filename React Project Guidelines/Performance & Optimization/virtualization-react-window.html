Virtualization – טעינה חכמה של רשימות גדולות עם React Window

← חזרה לדף הבית

למה צריך וירטואליזציה?

  • שיפור ביצועים משמעותי ברשימות גדולות.
  • פחות אלמנטים ב־DOM בכל רגע נתון.
  • גלילה חלקה גם עם אלפי פריטים.

מייצגים רק את הפריטים שנמצאים כרגע בתצוגה — לא את כל הרשימה.

React Window — הספרייה הפופולרית

ספרייה קלה שמרנדרת רק את הפריטים הגלויים, עם חישוב חכם של גודל ומיקום. 

import { FixedSizeList as List } from 'react-window';

const Row = ({ index, style }) => (
  <div style={style}>
    Row {index}
  </div>
);

export default function App() {
  return (
    <List
      height={300}
      itemCount={1000}
      itemSize={35}
      width={300}
    >
      {Row}
    </List>
  );
}

כאן מוצגים ב־DOM רק הפריטים שנראים לעין, לא כל ה־1000.

מתי להשתמש?

  • רשימות ענק (טבלאות, אינפיניטי סקרול).
  • פידים ארוכים של תוכן.
  • כמויות נתונים גדולות מה־API.

טיפים לשימוש חכם

  • בחרו בין FixedSizeList (גובה קבוע) ל־VariableSizeList (גובה משתנה).
  • שלבו עם אינפיניטי סקרול לטעינת נתונים נוספים בזמן גלילה.
  • שמרו על פריט פשוט — עיבוד כבד בתוך פריט יפגע ביתרון.

סיכום

Virtualization מאפשר להציג כמויות עצומות של נתונים בלי לפגוע בביצועים. עם React Window תקבלו פתרון קליל, יעיל ונוח לשילוב כמעט בכל רשימה.
