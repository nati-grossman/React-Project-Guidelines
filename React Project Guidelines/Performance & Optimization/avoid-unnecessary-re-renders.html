Avoid Unnecessary Re-renders – זיהוי וטיפול

← חזרה לדף הבית

למה זה קורה?

  • שינוי state גם כשהערך לא באמת השתנה.
  • פונקציות/אובייקטים נוצרים מחדש בכל רינדור.
  • props שזולגים לעומק העץ ומרנדרים ילדים.
  • עדכוני context שמרעננים את כל הצרכנים.

איך מזהים?

  • React DevTools – Highlight Updates: זיהוי ויזואלי בזמן אמת.
  • Console.log חכם: הדפסת props ושינויים ממוקדים.
  • Profiler: מדידת זמן רינדור לכל קומפוננטה.

איך מטפלים?

  • React.memo: מונע רינדור אם ה־props לא השתנו.
  • useCallback / useMemo: שומרים פונקציות/ערכים כדי לשמור reference יציב.
  • פיצול קומפוננטות: הפרדת חלקים סטטיים מדינמיים.
  • מיקוד state: שמירת state קרוב למקום הצריכה.
  • Context חכם: פיצול Context או שימוש ב־selectors כדי לצמצם רענונים.

טיפ חשוב

אל תוסיפו Memoization סתם. ודאו קודם שיש בעיית ביצועים אמיתית — אופטימיזציה מוקדמת מיותרת מסבכת את הקוד ופוגעת בקריאות.

סיכום

רינדורים מיותרים הם כמו פגישות מיותרות — מעייפים ולא מקדמים. עם הכלים והמתודולוגיה הנכונה, תשמרו על אפליקציה זריזה ומגיבה גם כשהיא גדלה.
