Fetch / Axios – עבודה עם REST APIs

← חזרה לדף הבית

למה Fetch / Axios חשובים?

בפיתוח React, כמעט תמיד נצטרך לתקשר עם שרתים כדי לשלוף או לעדכן נתונים. שימוש ב‑Fetch או Axios הוא הדרך הפופולרית לעשות זאת בצורה אסינכרונית ומבוקרת.

  • מאפשר לממשק המשתמש לקבל ולהציג מידע בזמן אמת.
  • מאפשר שליחה של נתונים מה‑UI ל‑backend בצורה מסודרת.
  • חיוני לניהול שינויים, CRUD operations ו‑state באפליקציה.
  • מספק בסיס לשילוב עם כלים לניהול state ו‑caching כמו React Query או SWR.

איך זה עובד

1. Fetch API

מובנה ב‑JavaScript, ללא צורך בהתקנות נוספות.

מבצע קריאות HTTP בצורה promise-based:

// GET request
async function fetchUsers() {
  try {
    const response = await fetch('/api/users');
    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`);
    }
    const data = await response.json();
    return data;
  } catch (error) {
    console.error('Fetch error:', error);
    throw error;
  }
}

// POST request
async function createUser(userData) {
  try {
    const response = await fetch('/api/users', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
      },
      body: JSON.stringify(userData),
    });
    
    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`);
    }
    
    const data = await response.json();
    return data;
  } catch (error) {
    console.error('Create user error:', error);
    throw error;
  }
}

2. Axios

ספרייה פופולרית עם יתרונות נוספים על Fetch:

  • טיפול אוטומטי ב‑JSON.
  • תמיכה ב‑interceptors ל‑requests ו‑responses.
  • ניהול שגיאות נוח יותר.
  • תמיכה מובנית ב‑timeout ו‑cancel requests.
import axios from 'axios';

// Create axios instance with default config
const api = axios.create({
  baseURL: '/api',
  timeout: 10000,
  headers: {
    'Content-Type': 'application/json',
  },
});

// GET request
async function fetchUsers() {
  try {
    const { data } = await api.get('/users');
    return data;
  } catch (error) {
    console.error('Axios error:', error);
    throw error;
  }
}

// POST request
async function createUser(userData) {
  try {
    const { data } = await api.post('/users', userData);
    return data;
  } catch (error) {
    console.error('Create user error:', error);
    throw error;
  }
}

// PUT request
async function updateUser(id, userData) {
  try {
    const { data } = await api.put(`/users/${id}`, userData);
    return data;
  } catch (error) {
    console.error('Update user error:', error);
    throw error;
  }
}

// DELETE request
async function deleteUser(id) {
  try {
    await api.delete(`/users/${id}`);
    return true;
  } catch (error) {
    console.error('Delete user error:', error);
    throw error;
  }
}

שימוש ב־React Components

import React, { useState, useEffect } from 'react';
import axios from 'axios';

function UsersList() {
  const [users, setUsers] = useState([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const fetchUsers = async () => {
      try {
        setLoading(true);
        const { data } = await axios.get('/api/users');
        setUsers(data);
      } catch (err) {
        setError(err.message);
      } finally {
        setLoading(false);
      }
    };

    fetchUsers();
  }, []);

  const handleAddUser = async (newUser) => {
    try {
      const { data } = await axios.post('/api/users', newUser);
      setUsers(prev => [...prev, data]);
    } catch (err) {
      setError(err.message);
    }
  };

  const handleDeleteUser = async (id) => {
    try {
      await axios.delete(`/api/users/${id}`);
      setUsers(prev => prev.filter(user => user.id !== id));
    } catch (err) {
      setError(err.message);
    }
  };

  if (loading) return <div>Loading...</div>;
  if (error) return <div>Error: {error}</div>;

  return (
    <div>
      <h2>Users</h2>
      <ul>
        {users.map(user => (
          <li key={user.id}>
            {user.name}
            <button onClick={() => handleDeleteUser(user.id)}>
              Delete
            </button>
          </li>
        ))}
      </ul>
    </div>
  );
}

Axios Interceptors – הגדרות מתקדמות

// Request interceptor - add auth token
api.interceptors.request.use(
  (config) => {
    const token = localStorage.getItem('token');
    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }
    return config;
  },
  (error) => {
    return Promise.reject(error);
  }
);

// Response interceptor - handle errors globally
api.interceptors.response.use(
  (response) => {
    return response;
  },
  (error) => {
    if (error.response?.status === 401) {
      // Unauthorized - redirect to login
      localStorage.removeItem('token');
      window.location.href = '/login';
    }
    
    if (error.response?.status >= 500) {
      // Server error - show notification
      console.error('Server error:', error.response.data);
    }
    
    return Promise.reject(error);
  }
);

טיפים למימוש יעיל

  • תמיד לטפל בשגיאות ולבדוק status או ok.
  • להשתמש ב‑async/await כדי לכתוב קוד קריא וברור.
  • למנוע קריאות חוזרות מיותרות עם caching או state management.
  • לשקול שימוש ב‑Axios אם רוצים interceptors, timeout, או ניהול cookies/session.
  • לשלב עם React Query / SWR למקסימום יעילות וניהול אוטומטי של cache.
  • להגדיר base URL ו‑default headers בצורה מרכזית.

סיכום

שימוש נכון ב‑Fetch או Axios מאפשר שליטה מלאה בקריאות API, טיפול בשגיאות וניהול נתונים באפליקציות React. שילוב עם כלים לניהול state ו‑caching מבטיח ביצועים טובים, UI מעודכן וחוויית משתמש אמינה.
